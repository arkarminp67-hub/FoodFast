# Lab 04: MVP Feature Prioritization List

[cite_start]Using MoSCoW mapping constraints, our development timeline focuses strictly on software logic that requires zero physical IoT dependencies[cite: 72]:

## 1. MUST HAVE (Core Semester Prototype Scope)
* **Digital Menu & Selection Form:** Interactive listing showing vendor items and item prices.
* **Simulated Token Generator:** Logic to process form actions and convert them into interactive digital receipts.
* **Real-time Order Status Bar:** A clean web dashboard interface displaying structural updates (`Pending` -> `Preparing` -> `Ready`).

## 2. SHOULD HAVE (Build if timeline permits)
* **Simulated QR Sandbox Payment:** Quick form interaction replicating a mobile wallet payment flow.

## 3. COULD HAVE (Future expansions)
* **Favorite Order Memory:** Cookie-based storage to remember standard student orders.

## 4. WON'T HAVE (Excluded due to IoT & security constraints)
* [cite_start]**Physical Smart Pickup Lockers:** Excluded because it requires hardware IoT engineering[cite: 72].
* [cite_start]**Live Credit Card Processing:** Excluded to avoid complex encryption and compliance overhead[cite: 72].
